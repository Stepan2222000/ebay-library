## URL выдачи и фильтры

URL строит `config.build_search_url(query, *, page, zip, condition, min_price,
max_price)` (импортируется из пакета). Структурные части всегда: `_nkw`
(запрос, URL-энкодится), `_ipg=240` (`ITEMS_PER_PAGE`, максимум eBay), `_pgn`
(страница, 1-based). Фильтры опциональны — добавляются только если заданы:

- `zip` → `_stpos`: индекс доставки. **Задаёт РЕАЛЬНУЮ локацию**, не косметику:
  доставка пересчитывается под ZIP (live: 19701 vs Аляска 99950 → разные суммы
  доставки и число результатов; меняется и лейбл «Shipping to <zip>»). Без ZIP
  у части карточек доставка «Shipping not specified» и нет location → парсер
  падает на `shipping_cost` ⇒ на практике **ZIP нужен всегда** (рекоменд.
  "19701"). `_fcid=1` (страна US) не обязателен — `_stpos` достаточно.
- `condition` → `LH_ItemCondition`: `"all"`/`None` — без фильтра; `"new"` = `3`
  (Brand New + New Other); `"used"` = `3000` (Used / Pre-Owned). Фактическое
  состояние карточки всё равно парсим.
- `min_price`/`max_price` → `_udlo`/`_udhi`: границы цены (USD). Подтверждено
  live, фильтруют реально.

ZIP — это URL-параметр, **UI-установки ZIP больше нет** (раньше был модал-флоу,
удалён). Прежнее «`_stpos` косметический» неверно для текущего eBay.

Замечание: вёрстка SRP и валюта зависят от **страны IP** (US-IP → каноническая
вёрстка + USD; не-US IP → локализованная вёрстка и местная валюта, URL-параметром
не переопределяется). Воркер ходит через US-прокси.

## Парсинг страницы (Слой 1, `parse_search_page`)

Возвращает `SearchPage` с `SrpCard` — сырьём «как на сайте» (цена/доставка в
исходной валюте + `currency_raw`), БЕЗ перевода в USD (это делает Слой 2, см.
ниже). Можно гонять офлайн на сохранённом HTML.

- Карточки до сепаратора «Results matching fewer words» — только точные;
  после — «похожие», не берём. Сепаратор встретился → `has_fewer_words_sep`.
- Placeholder «Shop on eBay» (data-listingid длиннее 12 цифр) — пропускаем.
- `results_count` — из счётчика; справочный (циклом пагинации не управляет).
- «0 results» — валидный пустой результат, не ошибка.
- `location` опционален: eBay рендерит «Located in» лениво и не всегда (даже
  при выставленном ZIP может отсутствовать у всей выдачи — подтверждено live
  и фикстурой `srp_8M0142836_zip19701.html`). Нет строки → None. Точный
  location при необходимости берём с item-страницы. Остальные поля
  обязательны → ParseError.

## Пагинация (`fetch_catalog`)

Цикл `_pgn=1,2,…` — стоп-сигналы, в порядке проверки:

1. **сепаратор fewer-words** — точные кончились;
2. **страница неполная** (< items_per_page карточек) — последняя.

⚠️ eBay **за концом не отдаёт пустую страницу** — клампит на последнюю
(подтверждено: `_pgn=3` при двух реальных = копия `_pgn=2`). Поэтому стоп
«по 0 карточек» не работает; стоп по «<240» не делает лишнего запроса.

Между соседними страницами выдача **нахлёстывается** (живая выдача сдвигается
между запросами; хвост N ≈ начало N+1, наблюдали 7–9 карточек) — дедуп по
`item_id` обязателен (первое вхождение побеждает). Счётчик может слегка
расходиться с фактом (±%) — это свойство eBay, ориентируемся на факт.

Любой сбой (блок/таймаут/ParseError) — подзадача падает целиком, без
частичных результатов.

## Конвертация цен в USD (`fx.convert_cards`)

Чистый парсер отдаёт `SrpCard` с ценой/доставкой в **исходной валюте** + сырым
токеном `currency_raw` (`'$'`, `'C $'`, `'EUR'`…). Перевод в USD — отдельный
шаг Слоя 2 на живом HTTP (парсер офлайн, курсов не знает):

- После сбора всех страниц `fetch_catalog` зовёт `fx.convert_cards(cards)` —
  **один батч на весь каталог** (не на страницу/карточку).
- Собираем уникальные `(сумма, валюта)` (цены + платные доставки), **дедуп**, и
  параллельно (без лимита, только пул httpx) дёргаем
  `GET {FX_API_URL}/convert?amount=&from=<токен>&to=USD`. Эндпоинт сам резолвит
  написание (`fx.currency_aliases`) и держит курсы в своём кэше (TTL).
- `Free` (0.0) и «не указана» (None) доставка не конвертируются — как есть.
- Итог — `CatalogItem` с `price`/`shipping_cost` **в USD**; исходную валюту в
  модели не храним.

`FX_API_URL` — env, дефолт `http://194.164.245.107:8092`. Без фолбеков:
неизвестная валюта (404) или недоступность сервиса (сеть/таймаут/5xx) → сбой
наружу (в блоке изолируется per-подзадача). Если eBay показал валюту, которой
нет в `fx.currency_aliases`, — повод дозалить алиас (см. `migration_005`).

Параметры `/rate` (курс на валюту) хватило бы на 1 запрос/страница, но выбран
`/convert` на сумму с дедупом и параллелизмом — воркеры ходят с серверов рядом
с fx-сервисом.

## Блок подзадач (`fetch_catalogs`)

- Вход: список запросов; обрабатываются по порядку на одном `page`.
- Выход `CatalogBatch`: объединённый `items` (глобальный дедуп по item_id по
  всему блоку), `per_query` (детали успешных), `errors` (упавшие).
- Упавшая подзадача НЕ валит блок — изолируется в `errors`, остальные
  продолжают; переотдача — забота оркестратора.
