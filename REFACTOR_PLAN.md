# План: новая реализация библиотеки (v2)

Большой рефакторинг Слоя 2 + улучшение структуры. Новая реализация собирается
**в соседней папке-пакете в этом же репозитории** — старый пакет `ebay_library/`
на время работ замораживается (не редактируется вообще) и служит источником
проверенного кода. Из старого переносится всё, что подтверждено живыми
прогонами: селекторы, алгоритмы парсинга, механика ZIP, пагинация, готовность.
Переписывается слой добычи (воркерные методы) — под новый контракт ошибок и
сессию с заменой страниц.

---

## 1. Зафиксированные решения (не обсуждаются повторно)

Политика ошибок:

| Ситуация | Действие |
|---|---|
| Pardon | ждём до хардкапа 180с; не прошёл → **критическое исключение** |
| Error Page («Sorry — Something went wrong») | **критическое исключение** сразу |
| Access Denied | **замена страницы**, продолжаем с того же места, без лимита попыток |
| Транспортная смерть (`net::ERR_*`, краш, закрытие page) | то же — замена страницы |
| ParseError, сбой fx, таймаут iframe-описания, всё неопознанное | **критическое исключение** |

Критическое исключение валит задачу и воркера целиком; частичных результатов
нет, переотдача — забота оркестратора. Машинерии «мёртвой сессии» нет — после
критического исключения воркер завершается, объект сессии умирает с ним.

Сессия и замена страниц:

- `EbaySession(get_page, *, page_delay_s=7.0)` — вместо готового `page`
  библиотека получает **async-колбек воркера** `get_page()` → свежий page.
  Старые страницы библиотека не закрывает — утилизирует воркер.
- Перед каждым replacement-запросом новой страницы — **фиксированная** пауза
  `page_delay_s` (дефолт 7с; не растёт, не сбрасывается); перед самым первым
  получением страницы паузы нет. Темп/лимит/учёт замен — точка контроля
  воркера внутри `get_page()` (может ждать сколько нужно или бросить
  исключение — оно критическое).
- Прогрев (заход на главную) делает сессия сама на каждой полученной странице;
  блок/смерть прямо на прогреве → следующая замена (бесконечно).
- Гранулярность продолжения: каталог — с упавшей страницы выдачи (собранные
  карточки не теряются, дедуп переживает замену); item — товар целиком.

Каталог:

- **Один универсальный** `fetch_catalog(queries, ...)`: принимает `str` или
  `list[str]`, всегда возвращает один тип результата — общий список карточек с
  глобальным дедупом по item_id (первое вхождение побеждает) + разбивка
  `per_query`. Поля `errors` больше нет (ошибки критичны и летят наружу).
- ZIP/condition/цена — через URL на каждой странице (как сейчас).

Поля (решения по live-находкам 2026-06-10):

- `seller`, `condition`, `shipping_cost` — **обязательны** (ParseError при
  отсутствии). Известно: в категориях Authenticity Guarantee (часы, кроссовки)
  eBay не рендерит продавца на SRP вовсе — задачи по ним будут падать;
  обработку добавим позже, когда понадобится.
- `shipping_cost` сужается до `float` (0.0 = Free): «Shipping not specified» →
  ParseError, значит `None` недостижим — тип не должен врать.
- `location` (SRP) и `last_updated` (item) — опциональны, как было.

fx: параллелизм остаётся **безлимитным** (решение пользователя; известно, что
сервис держит ~200–250 одновременных соединений — с серверов рядом с сервисом
пик короче).

## 2. Структура нового пакета

Рабочее имя пакета — `ebaylib/` (обсуждаемо, см. §8). Слои становятся явными:
три вида IO разнесены по подпакетам, модели и ошибки — наверху (это контракт
API, а не деталь парсинга).

```
ebaylib/
  __init__.py        # публичное API (см. §3)
  errors.py          # все исключения; чистый (без playwright-импортов)
  models.py          # dataclasses (frozen, slots) — контракт наружу
  urls.py            # build_search_url, ITEMS_PER_PAGE (бывш. config.py)
  html/              # Слой 1: чистый парсинг HTML/url → данные, без сети
    __init__.py
    selectors.py     # ВСЕ CSS-селекторы (единственное место)
    normalize.py     # нормализация condition
    page_state.py    # «что за страница» по url+title + antibot
    srp.py           # parse_search_page (бывш. parsing/catalog.py)
    item.py          # parse_item_page + ship_to_location (вобрал zipstate.py)
  http/              # HTTP-IO без браузера
    __init__.py
    fx.py            # convert_cards (конвертация валют в USD)
    images.py        # fetch_images (бывш. download.py)
  browser/           # Слой 2: живой Playwright
    __init__.py
    readiness.py     # wait_until_ready + якоря (бывш. worker/navigation.py)
    session.py       # EbaySession — НОВОЕ (вместо worker/fetch.py)
```

Чем лучше старой раскладки:

- `models.py`/`errors.py` наверху: потребитель видит контракт, не лазая в
  `parsing/`;
- имена подпакетов говорят про вид IO (`html` — ничего, `http` — сеть без
  браузера, `browser` — Playwright), а не про роль («worker» был расплывчат);
- `config.py` → `urls.py` (модуль строит URL — так и называется);
- `zipstate.py` влит в `html/item.py`: `ship_to_location` — часть item-флоу,
  отдельный файл на 30 строк не оправдан;
- классификация транспортных ошибок Playwright живёт в `browser/session.py`,
  а `errors.py` остаётся импортируемым без playwright (офлайн-парсинг на
  машине без браузера).

## 3. Публичное API

```python
from ebaylib import EbaySession, fetch_images, build_search_url
# Слой 1 доступен отдельно: parse_search_page, parse_item_page,
# ship_to_location, модели, page_state, ошибки.

async def get_page():            # реализует воркер: прокси/стелс/новый page
    ...

session = EbaySession(get_page)  # page_delay_s=7.0 по умолчанию

batch = await session.fetch_catalog(
    ["8M0142836", "861787"],     # или одна строка — API универсальный
    zip="19701", condition="new", min_price=50, max_price=500,
)
item = await session.fetch_item("277574984378", zip="19701")
photos = await fetch_images(item.image_urls)
```

Уходят из публичного API: `warmup` (внутрь сессии), `fetch_item` /
`fetch_catalog` / `fetch_catalogs` как свободные функции. Остаются:
весь Слой 1, `convert_cards`, `fetch_images`, `wait_until_ready`
(диагностика), модели, ошибки.

Переименование: `CatalogBatch` → `CatalogResult` (поле `errors` ушло, смысл
«блок с изоляцией ошибок» сместился к «итог универсального вызова») —
предварительно да, обсуждаемо.

## 4. Что переносится из старого как есть (проверено живьём)

| старое | новое | правки при переносе |
|---|---|---|
| `parsing/selectors.py` | `html/selectors.py` | нет |
| `parsing/normalize.py` | `html/normalize.py` | нет |
| `parsing/page_state.py` | `html/page_state.py` | только докстринг политик |
| `parsing/catalog.py` | `html/srp.py` | нет (логика 1:1) |
| `parsing/item.py` + `parsing/zipstate.py` | `html/item.py` | слияние файлов, логика 1:1 |
| `parsing/models.py` | `models.py` | `shipping_cost: float`; `CatalogBatch` → без `errors` (+рен.) |
| `config.py` | `urls.py` | нет |
| `fx.py` | `http/fx.py` | только докстринг (изоляции per-подзадача больше нет) |
| `download.py` | `http/images.py` | нет |
| `errors.py` | `errors.py` | только докстринги политик |
| `worker/navigation.py` | `browser/readiness.py` | только докстринги политик |
| Алгоритмы: пагинация со стопами (сепаратор / неполная страница), дедуп с нахлёстом, ZIP-флоу item (оптимистичный заход + setter-SRP + сверка `shipToLocation`), добыча iframe-описания со скроллом | `browser/session.py` | переносятся внутрь методов сессии 1:1 |

## 5. Что пишется заново: `browser/session.py`

Ядро (контур отработан прототипом в /tmp, все сценарии зелёные):

```
_page_dead(exc)   — транспортная смерть (снято живьём, Playwright 1.60):
                    Error, не TimeoutError; TargetClosedError (по имени типа —
                    из playwright.async_api он не экспортируется),
                    'net::ERR' либо 'Target crashed' в сообщении.
_retryable(exc)   — AccessDeniedError или _page_dead.

_acquire(replacement) — [пауза 7с если replacement] → get_page() → прогрев;
                    ретраябельный сбой прогрева → следующая итерация (∞).
_run(unit)        — лениво берёт первую страницу; unit(page) до успеха,
                    ретрай только на _retryable, прочее — наружу.

fetch_catalog     — цикл по запросам → цикл по страницам; unit = одна
                    страница выдачи; после страниц запроса — один fx-батч;
                    глобальный дедуп; → CatalogResult.
fetch_item        — unit = товар целиком (заход, ZIP-сверка/сеттер,
                    iframe-описание, парс) → ItemPage.
```

## 6. Тесты

- `tests/fixtures/` — без изменений (живой en-US HTML).
- Существующие `test_catalog.py` / `test_item.py` / `test_page_state.py` —
  переводятся на импорты v2; в `test_catalog` правка под `shipping_cost: float`.
- **Новый `test_session.py`** — контур замены на фейковых страницах (порт
  отработанного прототипа): продолжение с упавшей SRP-страницы без потери
  карточек; замена при блоке на прогреве; пауза перед каждой заменой и не
  перед первой; item целиком на новой странице; фатальные — сразу наружу без

## 8. Решено после обсуждения

1. **Пакеты**: новый — `ebaylib/` в этом же репо; старый `ebay_library/`
   заморожен на время работ; после живой валидации старый удаляем, новый
   переименовываем в `ebay_library` (имя импорта воркеров не меняется,
   `pyproject.toml` уже подходит — не трогаем до финала).
2. **`CatalogBatch` → `CatalogResult`** — да.
3. **Git**: снапшот-коммит текущего состояния до старта (точка отката),
   далее — коммит на каждый этап (§9).
4. **Логирование** (дефолт, можно отменить): стандартный
   `logging.getLogger("ebaylib")` — INFO на замене страницы (причина: denied /
   net::ERR / crash + место: запрос/страница/товар), DEBUG на прогрессе
   (прогрев прошёл, страница выдачи собрана). Без хендлера у воркера —
   молчит, как и раньше; сигнатура `get_page()` не меняется.
5. **Конструктор сессии** (дефолт): только `page_delay_s=7.0`;
   `desc_timeout_s` — параметр `fetch_item`; Pardon-хардкап (180с) и таймаут
   якорей (30с) — константы `readiness.py` (свойства eBay, не развёртывания).
6. **Дубликаты запросов** в списке `fetch_catalog` — дедуп на входе с
   сохранением порядка (повтор молча отбрасывается, на eBay не ходим дважды).
7. **Контракт `get_page`**: обязан отдавать свежую страницу (новый
   контекст/прокси) — документируем, не проверяем (вернёт ту же — замены
   зациклятся на том же блоке, это зона воркера).
8. Судьба этого файла: удаляется на финальном этапе (содержимое к тому
   моменту переедет в обновлённые спеки).

## 9. Этапы реализации

Каждый этап завершается терминальной проверкой и коммитом. Стоп-точки —
этапы 3, 4 и 6: после них отчёт и ожидание подтверждения.

**Этап 0 — снапшот.**
Коммит текущего состояния рабочей копии как точка отката (правки старого
пакета, спек, тестов + этот план). Ничего не пишем.
Проверка: чистый `git status`.

**Этап 1 — каркас, Слой 1, http.**
Создать `ebaylib/`: `errors.py`, `models.py` (`shipping_cost: float`,
`CatalogResult` без `errors`), `urls.py`, `html/*` (перенос 1:1, слияние
`zipstate` → `item`), `http/*`, `__init__.py` (экспорт Слоя 1).
Проверка: существующие фикстурные тесты, переведённые на импорты v2
(`test_catalog` — правка под `float`), зелёные; терминальный санити-прогон
парсеров на фикстурах.
Коммит: «v2: skeleton + layer 1 + http».

**Этап 2 — browser/: readiness + session.**
`browser/readiness.py` (перенос + докстринги политик),
`browser/session.py` (`EbaySession`: `_page_dead`/`_retryable`, `_acquire`,
`_run`, `fetch_catalog`, `fetch_item`, логирование §8.4).
Проверка: терминальный прогон ретрай-контура на фейковых страницах
(сценарии из /tmp-прототипа) против НАСТОЯЩЕЙ `EbaySession`.
Коммит: «v2: browser layer — readiness + EbaySession».

**Этап 3 — формальные тесты сессии.** ⏸ стоп-точка
`tests/test_session.py`: продолжение с упавшей SRP-страницы без потери
карточек; замена при блоке на прогреве; пауза перед каждой заменой и не
перед первой; item целиком на новой странице; фатальные наружу без
`get_page`; `_page_dead` на реальных playwright-исключениях; дедуп
дубль-запросов; универсальность входа (str | list).
Проверка: все тесты репо зелёные. Коммит: «v2: session tests».
→ Отчёт: что покрыто, что показали прогоны.

**Этап 4 — живой smoke.** ⏸ стоп-точка
Реальный Chrome с этой машины: каталог (1–2 небольших запроса, категория с
продавцом) + item через `EbaySession`. Оговорки: ParseError на «Shipping not
specified» — поведение by design (перепрогон другим запросом); fx безлимитен
— запросы небольшие, чтобы не упереться в локальную дальность до сервиса.
Коммит (если были фиксы). → Отчёт с цифрами прогона.

**Этап 5 — документация.**
`specs/architecture.md`, `page_readiness.md`, `catalog_flow.md`,
`item_flow.md`; `README.md` (пример с `EbaySession` + полный рабочий
`get_page`); `CLAUDE.md` (раскладка слоёв). Коммит: «v2: docs».

**Этап 6 — финал.** ⏸ только после явного подтверждения
Удалить `ebay_library/` (старый), переименовать `ebaylib/` → `ebay_library`,
поднять версию до 0.1.0, удалить `REFACTOR_PLAN.md`, финальный прогон всех
тестов + короткий живой smoke. Коммит: «v2 → ebay_library».
